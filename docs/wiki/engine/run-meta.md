# Run & meta (engine internals)

*Audit chunk 9. Page anatomy + durability rules: [[README|engine hub]]. The roguelike-structure concept companion is [[roguelike-meta]]; the reward-screen splice/clone mechanics live in [[synthesis-staple]], the opponent-scaling mechanics in [[draft]]. This page carries the verified mechanics of `run.js` and `picklog.js`: the slot model, the save/migration lifecycle, the mid-game snapshot revert, reward generation, the endless map, run modifiers, and the picklog analytics layer.*

## What it does

The run layer is everything that outlives a single game: the run deck (the **slots array**), the localStorage save with its versioned migration chain (`load`/`save`/`MIGRATIONS`), the mid-game-quit snapshot revert (`midGameSlotsSnapshot` / `rollbackForMidGameRestore`), the win-reward machinery (`generateRewardOffer` → `rollOneCandidate` → `pickRewardCandidate`), result recording (`recordResult`), the sector map (`generateMap`), the seven run-start boons (`RUN_MODIFIERS`), and a fire-and-forget pick-analytics sidecar (`PICKLOG.logPick` / `finishDraft`) that no gameplay code reads.

## The flow

- **The slot model is the run's spine.** The slots array *is* the run deck; every in-game card caches its run-slot index as `slotIdx` at deck build. That cache creates a documented invariant class: `removeSlotByIdx` carries a written **caller contract** — anyone removing a slot must decrement the cached `slotIdx` of every in-game card sitting past the removed index. The honoring sites are `ripSlotForPhylactery`, `ripSlotByIdx` (side-aware, does both halves), and both splice fixups; the audit enumerated the violators — the `EFFECTS.rip` (Vile Edict) handler skips the fixup entirely (A9-2; live via both boss decks), and the same class was already filed engine-side as A5-4. A **second index set the contract never named** — `playedSlotIdxs`, the per-game record of which slots were actually played, feeding the win-reward filters via `recordResult` — is remapped by *no* removal site, even the contract-honoring ones (A9-3). Insertion discipline is clean: `appendSlot`-style writes go at end/after-self only, so the stale-pointer class is **fully enumerated** (A9-2, A9-3, A5-4) and one shared fixup helper closes all of it.
- **The save lifecycle and the migration-authoring lesson.** Saves carry a `SAVE_VERSION`; `load` runs `MIGRATIONS` entries upward while the blob's version is behind, then applies a sticker-only normalization pass — nothing else repairs shape. The durable lesson from this chunk: **a migration must be written against the git-verified real historical shape, never a remembered or guessed one.** The v1→v2 tplId-rename migration was authored against a phantom shape — it "migrated" four fields that never existed in any persisted save and missed the three real v1 tplId carriers (`midGameSlotsSnapshot`, `pendingReward.replacementPack` in both shapes, `modifier`) — and its unit test *pinned the phantom shape*, staying green while the migration was wrong: a test worse than absent (A9-1; downstream the un-migrated snapshot resurrects dead tplIds, the next deck build throws "Unknown card," and the catch wipes the save — total run destruction). The audit recovered the real v1 shape from git (`df2fd38^`), which is what the fix re-pins against. Companion discipline, same family: **every extension of a rename table needs a version bump** — the snake_case sweep (`e2e151f`) extended TPLID_RENAMES without bumping SAVE_VERSION, leaving a window of v2 saves no migration ever touches (A9-10). The migration loop is also upward-only: a future-version blob sails through unguarded (A9-4).
- **The mid-game snapshot revert (anti-farming, by design).** At game start the run takes `midGameSlotsSnapshot`, a full copy of the slots array; quitting mid-battle and resuming triggers `rollbackForMidGameRestore`, which restores the snapshot — you cannot farm mid-game rewards/slot changes by quitting. The audit verified the revert is **complete for run-state**: the only thing that escapes it is picklog's games-played counter (A9-5, analytics-only).
- **Rewards: pre-rolled at offer, persisted — the one designed randomness discipline.** `generateRewardOffer` rolls every candidate (`rollOneCandidate`) at *offer* time and persists the offer in the save; `pickRewardCandidate` only commits a pre-rolled choice. Reloading therefore re-presents the identical offer — **no reload-scum vector exists anywhere in the run layer** (everything is pre-rolled+persisted or committed-in-pick, audit-verified). The reward-type distribution `REWARD_TYPE_WEIGHTS` matches canon [[1504-rewards|§1504]] exactly. The guard *strength* across the four pick handlers is asymmetric (sticker/ripUp trust the pre-roll; twoStickers/transform re-validate) — harmless today because nothing mutates slots between offer and pick, parked with a pointer (A9-8). `recordResult` persists the game outcome plus `lastPlayedSlotIdxs`, the played-slots filter the sticker rewards aim at (the A9-3 invariant's consumer).
- **The endless sector structure.** `generateMap` builds one sector's node map; clearing the sector generates a fresh map and the run continues — **the run ends only in a loss**. Every map exit is a boss. Opponent difficulty scales with cumulative game number under the **/1 /3 /5** divisors (stickers / staples / clones, matching canon §1503); the scaling mechanics themselves are [[draft]]'s territory. Canon §1500 lagged the code on the endless structure and two reward wordings (A9-7).
- **The seven RUN_MODIFIERS.** Run-start boons, all seven traced end-to-end: every one returns *extras* (cards appended to the deck) — none mutates slots in place. The written contract was **contradictory** until the chunk-9 ship round: one comment layer declared boons pure, the other invited in-place mutation, and both the code comment and canon §1505 cited a phantom boon ("Watcher's Gift") that never existed — each layer citing the other's ghost (A9-6/A9-7; the adopted contract is mutation-allowed, matching what the call site tolerates).
- **Picklog is fire-and-forget analytics.** `PICKLOG.logPick` records draft picks (it receives the pre-mutation pack), `finishDraft` seals the record; double `finishDraft` no-ops and a malformed blob catch-resets — idempotent by inspection. No gameplay code reads it. Its one sharp edge is durable: picklog **re-applies TPLID_RENAMES unconditionally on every load**, so the rename table's ~250 legacy keys must never be reused as live card ids — a future card reusing one would have its picklog rows silently rewritten forever. The invariant holds today (key-set ∩ live cards = ∅) but nothing enforces it (A9-9, parked; the cheap assertion is the guard).

## Design rulings

No PR #98 mechanics rulings have landed for this subsystem yet — the staged decision packets (A9-1/A9-2/A9-3/A9-10) are queued for Joe's ballot; see the ledger. The campaign-level postures govern meanwhile: the global terminology ruling (Joe, 2026-06-10, PR #98 verdict rounds) — Magiclike's rules are **the** rules; and the remediation class rule — anything touching run-save load behavior or slot survival is game-outcome-altering, so it *stages* rather than ships on green. In-code deliberate designs the audit confirmed as design, not defect: the **mid-game snapshot revert** is intentionally anti-farming and intentionally complete for run-state; **pre-rolled persisted rewards** are the deliberate anti-reload-scum discipline; and the **endless sector structure** (the run ends only in a loss) is the code's intent — the rulebook lagged it, and A9-7 moves the docs to match the code.

## Verified clean

The audit checked and explicitly cleared: **REWARD_TYPE_WEIGHTS = §1504 exactly** (table-vs-canon diff clean); **opponent scaling matches §1503** (/1 /3 /5); **STICKER_ID_RENAMES values all exist** in the live sticker registry; **TPLID_RENAMES keys ∩ live cards = ∅ today** (the A9-9 invariant holds, just unguarded); **no reload-scum vector** — everything pre-rolled+persisted or committed-in-pick; **non-stackable sticker dedup holds** through the 2×/3× reward loops; **midGameSlotsSnapshot revert complete for run-state** (sole escape: the picklog counter, A9-5); **map integrity** — every node has ≥1 in-edge, every non-final node ≥1 out-edge; the legacy multi-successor warn-path is unreachable (back-compat only); **no-reward-on-win is impossible** with ≥1 slot; **zero AI call sites into RUN** re-confirmed; **slot-array insert discipline** (end/after-self only) means the stale-pointer class is fully enumerated; **picklog idempotency** by inspection. Cross-chunk concurrences: `RUN.applySplice` is reward-screen-only — the engine never calls it, so the in-game splice path stays [[synthesis-staple]]'s verified ground; the §1504 Clone wording mismatch is A5-11's, cited not re-filed.

## Coverage reality

As of the chunk-9 distillation (2026-06-11): the deep-read covered `run.js` and `picklog.js` **end-to-end (both files in full)**, all seven RUN_MODIFIERS traced, a complete RUN.* call-site sweep outside run.js, git bracketing of both migration boundaries (`df2fd38` for v1/v2, `e2e151f` for the rename sweep), and six executed probes on the scratch localStorage shim (migration round-trips, the edict and phylactery slot chains, the future-version blob, the crash-restore double-count, the rename-key intersection). The test fence is **thin and partly hostile**: run.js sits at ~17% on the (part-stale) A4-18 mutation map, and the touched region includes the cautionary example — `tplid_renames_test.js` *actively pinned the phantom migration shape*, certifying the A9-1 bug as correct: worse than no test. **Fully dark:** the entire reward pipeline (offer/roll/pick for all seven kinds), `load`'s normalization, the snapshot rollback, and sector clear. picklog.js is ~4% with **zero dedicated tests** — its IIFE caches state with no reset hook, which is the prerequisite for any real test there. The standing recommendation: a **migration round-trip battery pinned to the real git-derived v1 shape** (the A9-1 re-pin) plus a **slot-removal invariant test asserting both card slotIdx and playedSlotIdxs** across a rip — pinning A9-2 and A9-3 at once.
